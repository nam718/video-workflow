# 视频创作工作流

> 从剧本到分镜，再到关键帧和视频生成的一体化桌面工作流。

[![Release](https://img.shields.io/github/v/release/nam718/video-workflow)](https://github.com/nam718/video-workflow/releases)
[![License](https://img.shields.io/github/license/nam718/video-workflow)](./LICENSE)
[![Platform](https://img.shields.io/badge/platform-Windows%20%7C%20macOS-blue)](https://github.com/nam718/video-workflow/releases)

这是一个基于 Electron 的本地桌面应用，面向“剧本可视化”和“短视频批量生产”场景。它把传统上分散在多个工具中的流程串成一个完整闭环：导入剧本、提取角色场景、生成参考图、拆分分镜、推理提示词、调用视频服务生成素材，最后在时间轴里统一预览和挑选版本。

## 为什么做这个项目

很多 AI 视频工具只解决“单次生成”，但真正做内容时，麻烦通常出在流程衔接上：

- 剧本拆分和画面规划不连贯
- 角色、场景、道具的一致性很难维持
- 分镜修改后，提示词、图片和视频要反复手动同步
- 多账号、多任务批量生成时容易失控

这个项目的目标，就是把这些碎片化操作收拢到一个本地工作台里。

## 它能做什么

- 导入剧本或文案，统一设置视觉风格
- 自动提取角色、场景、道具
- 生成并管理参考图资产
- 将剧本拆分成可编辑的分镜
- 批量推理每个分镜的提示词
- 调用即梦、可灵等服务生成视频
- 支持并行生成、多账号多开并发
- 支持时间轴预览和多版本对比

## 工作流

```text
导入剧本
  -> 提取元素
  -> 生成参考图
  -> 剧本转分镜
  -> 推理提示词
  -> 生成关键帧 / 生成视频
  -> 时间轴预览
```

每一步结果都会保存在本地项目中，方便回退、修改和重复生成。

## 核心亮点

### 1. 一条线跑完整个视频生产流程

不是单点工具，而是完整工作流。你可以从最开始的文本剧本一路走到最终视频预览，中间所有节点都能编辑、重跑和复用。

### 2. 参考图驱动的一致性生成

角色、场景、道具会先被提取和图片化，再进入后续分镜和视频阶段，能明显减少“同一角色前后长得不一样”的问题。

### 3. 批量视频生成

支持普通并行生成，也支持多账号多开并发，把大批量视频任务交给多个浏览器实例处理。

### 4. 面向真实使用的项目管理

项目数据本地持久化，不是一次性对话产物。你可以随时关闭、重开、继续做，适合长流程创作。

## 项目结构

```text
electron/   Electron 主进程和本地 HTTP 服务
src/        前端页面
shared/     共享脚本、默认提示词和项目工具
scripts/    剧本分析、提示词生成等流程脚本
jimeng/     即梦集成
kling/      可灵集成
```

## 浏览器与构建说明

项目在运行时会优先尝试以下浏览器来源：

1. 打包时附带的 Chromium
2. 系统已安装的 Chrome / Edge

出于体积和版权分发考虑，开源仓库默认不提交浏览器二进制文件。如果你需要在自己的打包产物中内置浏览器，需要在本地自行准备对应目录后再构建。

## 仓库里刻意没有放什么

为了安全、体积和可公开分发，以下内容默认不会进入开源仓库：

- 打包产物，如 `dist/`
- 本地 Cookie、登录态和项目历史
- 本地测试目录和临时文件
- 浏览器二进制资源
- 与主应用无关的实验目录和旁支工具
