# 即梦，可灵精品工作流

> 一个把“剧本拆解、分镜规划、参考图生成、视频批量生产、时间轴预览”串成闭环的桌面工作台。

## 项目定位

`即梦，可灵精品工作流` 不是单点的 AI 生成工具，而是一个本地桌面端的视频生产流程系统。

它面向的不是“生成一张图”或“试一次视频”，而是更完整的创作链路：

- 从剧本文案开始
- 抽取角色、场景、道具
- 建立参考图资产
- 拆成可编辑的分镜
- 为每个分镜推理提示词
- 批量提交图片和视频生成
- 在时间轴里统一预览与挑选结果

如果你做的是小说可视化、短视频内容生产、广告样片、剧情类 AI 视频，这个项目解决的是“流程串联”和“一致性管理”的问题。

## 为什么这个项目有价值

很多 AI 视频工具的问题，不是模型不够强，而是流程太碎：

- 剧本和分镜脱节
- 角色设定前后不一致
- 分镜修改后，要重新手动同步提示词和素材
- 大批量视频任务提交后缺乏统一管理
- 多账号并发时切换混乱，难以持续生产

这个项目把这些步骤收束到一个桌面应用里，让它更像“创作工作台”，而不是一堆分散脚本。

## 核心能力

### 1. 剧本到分镜

- 支持导入 `.txt`、`.md` 文本
- 自动拆分剧情内容
- 转换成可编辑分镜
- 每个分镜都可以继续人工调整

### 2. 参考图资产管理

- 提取角色、场景、道具
- 为不同元素生成参考图
- 后续分镜和视频阶段自动引用这些素材
- 降低角色漂移、场景失真和风格不统一的问题

### 3. 批量提示词推理

- 为每个分镜生成视频提示词
- 支持自定义提示词预设
- 适合不同题材、风格和生成模型切换

### 4. 视频批量生成

- 支持并行提交视频任务
- 支持多账号多开并发
- 支持多模型组合生成
- 支持同一分镜多版本保存与回看

### 5. 时间轴预览

- 将生成结果统一落到时间轴
- 支持连续浏览
- 支持版本对比
- 更适合判断节奏、衔接和整体完成度

## 工作流结构

```text
导入剧本
  -> 提取角色 / 场景 / 道具
  -> 生成参考图
  -> 剧本转分镜
  -> 推理提示词
  -> 生成关键帧 / 生成视频
  -> 时间轴预览
```

这是一个逐层推进的流程，不是一次性生成。前一步结果会继续喂给下一步，因此更适合长链路创作。

## 适合的场景

- 网文、短篇故事、小说可视化
- 抖音、快手、B站等短视频内容生产
- 广告创意样片快速出片
- 教学演示和剧情化表达
- 多账号批量跑视频素材的工作室场景

## 开源仓库里不包含什么

为了让仓库适合公开分发，以下内容默认不会提交：

- 打包产物，如 `dist/`
- 本地 Cookie、登录态、项目历史
- 浏览器二进制文件
- 本地测试目录和临时数据
- 与主应用无关的实验性旁支目录

## 浏览器说明

项目运行时会优先尝试：

1. 打包时附带的浏览器
2. 系统本地已安装的 Chrome / Edge

开源仓库本身不携带浏览器二进制。如果你需要在自己的构建中内置浏览器，需要在本地准备对应资源后再打包。
