# 排行榜视频「智能解说」优化方案

> 问题发现日期: 2026-04-04  
> 涉及模块: `hongguo/ai-video-maker.mjs` → `buildScript()`  
> 状态: ✅ 已实现（2026-04-04）— 方案C（解说猫模式）

---

## 一、问题描述

生成的抖音 AI 短剧排行榜视频中，**解说词无法真正「解说」视频内容**。

当前解说效果（实际输出）：

> 冠军！白雪公主。作者小文。发布仅 1 天，经典 IP 改编是流量保障，分享率极高说明传播力强。

观众听到的全部是：**剧名 → 作者 → 天数 → 标签 → 数据播报**。  
没有告诉观众「这部剧讲了什么」「画面里在演什么」「为什么值得看」。  
画面与解说完全脱节——画面在播短剧片段，嘴里只在念数字。

---

## 二、根因分析

### 2.1 数据源没有剧情信息

抖音搜索/爬取得到的排行数据结构如下：

```json
{
  "rank": 1,
  "title": "这才是成年人该看的白雪公主...",
  "nickname": "小文",
  "aweme_id": "7526967501926780196",
  "liked_count": 4259272,
  "share_count": 2476680,
  "collected_count": 506729,
  "comment_count": 166124,
  "heat_score": 12951956,
  "create_time": 1752508705
}
```

**关键缺失**：没有 `description`（剧情简介）、没有 `category`（类型标签）、没有任何关于视频内容的文字。抖音短视频天然不提供这些字段。

### 2.2 解说文案靠模板硬拼

`buildScript()` 函数（ai-video-maker.mjs 第 117 行）完全基于结构化数据填模板：

```javascript
// 当前的解说模板
`${ordinals[i]}！${shortTitle}。作者${it.nickname}。${daysNote}${episodeNote}${featureNote}${dataNote}`
```

其中：
- `featureNote`：仅靠标题关键词匹配（`/古风/` → "古风赛道持续火热"），覆盖有限且千篇一律
- `dataNote`：纯数据播报（"分享率极高说明传播力强"）
- **没有任何关于视频实际内容的描述**

### 2.3 视频素材已下载但未利用

`clips_ai/` 目录已有 TOP10 的完整视频文件：

```
clips_ai/
  top01_7526967501926780196.mp4
  top02_7616594018055979195.mp4
  ...
  top10_7601744143487864104.mp4
```

这些视频含有对白、旁白、音效等音频信息，但从未被分析过。

---

## 三、解决方案：Whisper + LLM 自动内容解析

### 3.1 方案概览

```
已下载视频 (clips_ai/topXX_xxx.mp4)
    ↓ ffmpeg 提取音频
    ↓ Whisper 语音转文字（本地，已安装）
    ↓ 获得对白/旁白文本
    ↓ 连同 title 发给 LLM（已配置 gemini-2.5-flash）
    ↓ LLM 返回结构化内容摘要
    ↓ 嵌入 buildScript() 文案模板
```

### 3.2 已具备的基础设施

| 组件 | 状态 | 位置/版本 |
|------|------|----------|
| Whisper | ✅ 已安装 | `openai-whisper 20250625` (`/opt/homebrew/bin/whisper`) |
| LLM API | ✅ 已配置 | `gemini-2.5-flash` via `shared/call-ai.mjs` |
| FFmpeg | ✅ 已安装 | `/opt/homebrew/bin/ffmpeg` |
| 视频素材 | ✅ 已下载 | `hongguo/clips_ai/topXX_xxx.mp4` |

### 3.3 实现步骤

#### Step 1: 音频转文字（Whisper）

```bash
# 对每个 TOP10 视频提取音频并转写
ffmpeg -i clips_ai/top01_xxx.mp4 -vn -acodec pcm_s16le -ar 16000 /tmp/top01.wav
whisper /tmp/top01.wav --language zh --model small --output_format txt
```

- 预计每个视频 30-90 秒，Whisper small 模型本地处理约 10-20 秒/个
- 总计约 2-3 分钟完成全部 TOP10

#### Step 2: LLM 内容摘要

```javascript
// 将转写文本 + 标题发给 LLM
const prompt = `
你是短剧排行榜的视频解说员。根据以下信息，用一句话（30-50字）描述这部短剧的核心内容和看点。

视频标题: ${title}
视频对白/旁白:
${whisperText}

要求：
- 告诉观众这部剧讲了什么故事
- 点出最大看点或冲突
- 不要提及数据（点赞、分享等）
- 语气生动，让人想看
`;
```

- 一次批量请求 10 条，约 5-10 秒

#### Step 3: 改造 buildScript()

```javascript
// 改造后的解说模板
`${ordinals[i]}！${shortTitle}。${contentSummary}。作者${it.nickname}，${daysNote}${dataNote}`

// 改造后效果示例：
// "冠军！白雪公主。这版暗黑童话里公主不再等待拯救，
//  而是亲手拿起武器对抗邪恶皇后。作者小文，发布仅1天，
//  分享量高达247万，传播力极强。"
```

### 3.4 效果对比

| 维度 | 改造前 | 改造后 |
|------|--------|--------|
| 内容感 | ❌ 纯数据播报 | ✅ 有故事梗概 |
| 画面关联 | ❌ 画面与解说脱节 | ✅ 解说配合画面内容 |
| 观众体验 | 😴 像在听报表 | 🤩 像在看推荐 |
| 自动化 | ✅ 全自动 | ✅ 依然全自动 |
| 额外耗时 | - | +3-5 分钟 |

---

## 四、回退方案（无 Whisper 场景）

如果视频没有音频（纯音乐/无对白），可退化为：

1. **提取关键帧 → 视觉模型分析**：截取视频关键帧，发给多模态 LLM（GPT-4V/Gemini Vision）描述画面
2. **纯标题扩写**：仅基于标题让 LLM 猜测剧情方向（准确度较低但比纯数据好）
