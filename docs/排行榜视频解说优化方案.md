> 最后更新: 2026-04-04  
> 核心模块: `hongguo/3-make-video.mjs`  
> 状态: ✅ 已实现并持续迭代

---

## 一、系统概览

自动生成抖音 AI 短剧热度排行榜竖屏视频（1080×1920），包含：

- TOP10 排名片段（原视频 + AI 解说 + 字幕）
- 4 个 Remotion 数据动画图表（全屏展示）
- 开场快剪蒙太奇
- 背景音乐 + 音效
- 1.5 倍速版本

## 二、技术栈

| 组件 | 版本/位置 | 用途 |
|------|----------|------|
| Node.js | v20+ | 主脚本运行环境 |
| FFmpeg | `/opt/homebrew/bin/ffmpeg` | 音视频合成、拼接、字幕烧录 |

| edge-tts | `zh-CN-YunxiNeural` | 文字转语音 |
| Remotion | v4.0.441 (`hongguo/remotion/`) | React 数据动画图表渲染 |

## 四、核心流程

```
1. loadRanking()          加载 data/ 目录中最新排行 JSON
       ↓
2. analyzeAllClips()      Whisper 音频转写 + Gemini Vision 关键帧分析
       ↓
3. buildSmartScript()     LLM 生成 AI 解说文案（失败时回退到模板 buildScript()）
       ↓
4. renderRemotionCharts() Remotion 渲染 4 个数据动画视频（全屏 1080×1920）
       ↓
5. generateTTS()          edge-tts 语音合成（图表段用 atempo 拉伸匹配视频时长）
       ↓
6. compositeAll()         逐段合成：排名段 / 图表段 / 文字段 / 开场蒙太奇
       ↓
7. concatFinal()          FFmpeg concat 拼接所有段落
       ↓
8. addSfx()               叠加背景音乐和转场音效
       ↓
9. makeSpeedVersion()     生成 1.5 倍速版本
```

## 六、关键设计

### 6.1 AI 解说（方案C：Whisper + Gemini Vision）

**问题**：排行数据只有标题/点赞数等结构化字段，无剧情信息。纯模板解说像在念报表。

**解决**：
1. `clip-analyzer.mjs` 提取音频 → Whisper 转写对白
2. 提取关键帧 → Gemini Vision 分析画面内容
3. 将转写文本 + 画面描述 + 数据发给 LLM，生成自然解说文案
4. AI 失败时自动回退到模板解说 `buildScript()`

### 6.2 图表段 TTS 时长匹配

图表动画时长由 Remotion 决定（14-17 秒），TTS 时长可能不同。  
使用 FFmpeg `atempo` 滤镜将 TTS 音频拉伸/压缩到与图表视频等长：

```
tempo = clamp(ttsDur / chartDur, 0.5, 2.0)
```

### 6.3 Remotion 图表

4 个 React 组件渲染为 1080×1920 @ 30fps 的 MP4 视频：

| 图表 | 组件 | 时长 | 内容 |
|------|------|------|------|
| TOP10 排名 | `DouyinTop10Ranking` | ~15s | 柱状图排名动画 |
| 黑马飙升榜 | `DouyinDarkHorse` | ~16s | 日均增速排名 |
| 赛道热力图 | `DouyinTrackStrength` | ~17s | 各赛道热度对比 |
| 红海蓝海矩阵 | `DouyinOceanMatrix` | ~14s | 竞争度 vs 热度散点图 |

数据通过 `sync-ranking-data.mjs` 从排行 JSON 同步到 TypeScript。
